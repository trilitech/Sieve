LLM Configuration

Configure which LLM connection to use for AI script generation.

Choose an HTTP proxy connection pointing to an LLM API (e.g., Anthropic, OpenAI).

Select a connection first to load available models.

Maximum number of tokens for the LLM to generate.

Interpreters a script-type policy is permitted to invoke. Default /opt/sieve-py/bin/python3 matches the bundled Python venv. Symlinks are resolved before comparison so an attacker cannot bypass the list by symlinking into an allowed name. Stored policies that reference a disallowed command will fail at evaluation with a clear error.

TLS (optional)

Configure per-listener cert + key paths to serve HTTPS. Both fields in a pair must be set; leaving both empty keeps the listener on plaintext HTTP (the default for loopback). HSTS is set automatically on TLS responses.

Changes take effect after restart.

Externally-visible base URL used when Sieve constructs OAuth callback / redirect / setup URLs (GitHub App manifest, Google OAuth, Slack install). Default http://127.0.0.1:19816 matches the documented loopback production binding. Set this to your reverse-proxy URL if the admin UI is reached at a different address — never relied on inbound Host headers.

Security

Rotate the encryption passphrase that protects stored credentials. Sieve does not need to be restarted; agent calls in flight during the rotation receive a transient retry-safe error.
